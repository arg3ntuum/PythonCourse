1. Створення класу з інкапсуляцією
Створіть клас BankAccount, який має приватні атрибути для зберігання балансу та імені власника. Реалізуйте методи для отримання та зміни балансу, а також перевірку, чи можливо провести транзакцію, якщо баланс не є негативним.

2. Обмеження доступу до атрибутів
Створіть клас Person, в якому є приватні атрибути для віку та імені. Реалізуйте методи для отримання та зміни цих атрибутів через публічні методи, при цьому обмежте можливість змінювати вік, якщо він менший за 0 або більше 150 років.

3. Модифікація даних через геттер та сеттер
Створіть клас Rectangle, який має приватні атрибути для ширини та висоти. Реалізуйте геттер і сеттер для цих атрибутів, щоб можна було змінювати тільки позитивні значення для ширини та висоти.

4. Інкапсуляція в класі для збереження паролю
Створіть клас User, який зберігає пароль як приватний атрибут. Реалізуйте метод для встановлення паролю, який перевіряє, чи відповідає пароль мінімальним вимогам (наприклад, довжина паролю має бути не менша за 8 символів).

5. Обмеження доступу до методів за допомогою інкапсуляції
Створіть клас Employee, який має приватний метод для обчислення зарплати. Метод для отримання зарплати має бути публічним і викликати приватний метод для обчислення. Обмежте доступ до приватного методу лише зсередини класу.
